# 基础SQL考核（DDL/DML/查询）

要求：每题写出 SQL，并说明关键点（可写在你自己的笔记里）。

1. 创建一张 `coupons` 表：字段 `id`(主键自增)、`code`(唯一)、`discount_cents`、`status`、`created_at`。
2. 向 `users` 表新增一个用户 `eric`。
3. 查询 `users` 表中 `status=1` 的用户列表，按 `created_at` 倒序取前 2 条。
4. 查询每个用户的订单数（包含没有订单的用户）。
5. 查询 2025-01-01 ~ 2025-01-07 期间每天的订单量（按天聚合）。
6. 查询订单明细：订单ID、用户名、订单状态、订单金额、商品SKU、商品数量。
7. 查询“至少下过 2 单”的用户。
8. 更新：把 `orders` 表中状态为 `CREATED` 且创建时间早于 `2025-01-06 12:00:00` 的订单改为 `CANCELED`。
9. 删除：删除 `users` 中 `status=0` 且没有订单的用户。
10. 写出一个你认为合理的 `orders` 业务查询（自拟），并说明你会如何设计索引。
