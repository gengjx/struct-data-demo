# 性能与容量规划必须掌握知识点（对应 assessment 吞吐/分区规划）

## 1. Kafka 为什么吞吐高

### 概念

- 以日志为核心的追加写模型。

### 必会要点

- 顺序写磁盘 + 分段文件。
- Producer 批量发送与压缩。
- 发送与拉取的高效网络传输（理解为“减少拷贝与系统调用开销”的方向）。

### 对应考核

- 知识题 8：Kafka 为什么吞吐高。

## 2. 分区数规划

### 必会要点

- 并行度：分区数决定消费并行上限。
- 热点：key 设计决定负载是否均衡。
- 成本：分区过多会增加文件句柄、内存、控制面开销。

### 对应考核

- 知识题 9：分区数如何规划。

## 3. 基础容量估算（入门级）

### 必会要点

- 峰值写入速率（MB/s）
- 保留时间（retention）
- 副本数（replication factor）
- 由此估算磁盘容量与网络带宽
