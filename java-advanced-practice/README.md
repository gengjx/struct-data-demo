# Java进阶阶段学习练习

本模块包含了Java进阶阶段（第二部分）的完整练习代码，涵盖JVM深入、并发编程深入和设计模式。

## 📚 学习路线

### 第一部分：JVM深入
- **JVMPractice.java** - JVM深入练习
  - JVM内存模型（堆、栈、方法区、程序计数器、本地方法栈）
  - 垃圾回收机制（GC算法、GC收集器）
  - 类加载机制（加载、验证、准备、解析、初始化）
  - JVM调优参数和工具使用
  - 对象生命周期和内存泄漏分析

### 第二部分：并发编程深入
- **ConcurrentProgrammingPractice.java** - 并发编程深入练习
  - volatile关键字的深入理解（可见性、有序性）
  - synchronized的底层原理（对象头、锁升级）
  - CAS（Compare-And-Swap）机制和ABA问题
  - AQS（AbstractQueuedSynchronizer）框架
  - 线程池原理和自定义实现
  - 并发集合的使用（ConcurrentHashMap、BlockingQueue等）
  - Fork/Join框架
  - 死锁和死锁检测
  - ThreadLocal深入理解

### 第三部分：设计模式
- **DesignPatternPractice.java** - 设计模式练习
  - **创建型模式**
    - 单例模式（多种实现方式）
    - 工厂模式（简单工厂、工厂方法、抽象工厂）
    - 建造者模式
  - **结构型模式**
    - 代理模式（静态代理、动态代理）
    - 适配器模式
    - 装饰器模式
    - 组合模式
  - **行为型模式**
    - 观察者模式
    - 策略模式
    - 责任链模式
    - 模板方法模式
    - 状态模式
    - 命令模式

## 🚀 使用方法

1. **理论学习**
   - 先阅读相关理论知识（推荐《深入理解Java虚拟机》）
   - 理解每个知识点的原理和应用场景

2. **代码实践**
   - 打开对应的练习类
   - 阅读TODO注释，理解要求
   - 实现相应的代码

3. **源码阅读**
   - 阅读JDK源码（特别是并发包和集合类）
   - 理解底层实现原理

4. **工具使用**
   - 学习使用JVM调优工具（jmap、jstack、jvisualvm）
   - 分析内存使用情况和线程状态

## 📝 学习建议

### JVM部分
1. **理解内存模型**
   - 重点理解堆、栈、方法区的作用
   - 理解对象在堆中的分配过程

2. **理解GC机制**
   - 掌握各种GC算法的特点
   - 了解不同GC收集器的适用场景

3. **实践调优**
   - 使用JVM参数进行调优实践
   - 分析GC日志

### 并发编程部分
1. **深入理解原理**
   - 理解volatile、synchronized的底层实现
   - 理解CAS和AQS的工作原理

2. **阅读源码**
   - 阅读java.util.concurrent包的源码
   - 理解线程池的实现原理

3. **实践应用**
   - 在实际项目中应用并发编程技术
   - 注意线程安全和性能平衡

### 设计模式部分
1. **理解模式**
   - 理解每个模式解决的问题
   - 理解模式的结构和关系

2. **实践应用**
   - 在实际项目中应用设计模式
   - 避免过度设计

3. **源码学习**
   - 学习JDK和框架中的设计模式应用
   - 例如：JDK中的单例、工厂模式应用

## ⚠️ 注意事项

1. **JVM练习**
   - 某些练习可能导致程序异常，注意保存工作
   - 建议在测试环境中运行内存溢出相关练习

2. **并发编程练习**
   - 注意线程安全问题
   - 多线程程序可能难以调试，注意日志输出

3. **设计模式练习**
   - 不要过度使用设计模式
   - 理解模式的应用场景很重要

4. **源码阅读**
   - 建议使用IDE的单步调试功能
   - 结合官方文档理解API设计
